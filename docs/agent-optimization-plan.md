# Family Vault Agent 机制优化 - 开发计划

## 1. 现状诊断

### 当前流程
```
用户Query → Router LLM (调用#1) → 检索/处理 → Synthesizer LLM (调用#2) → 响应
```

### 核心问题
| 问题 | 影响 |
|------|------|
| 强制2次LLM调用 | 延迟↑ 成本↑（云端模型双倍计费） |
| 无智能缓存 | 相同查询重复计算 |
| 模型选择单一 | Router和Synthesizer用同等级模型，浪费 |
| 非流式输出 | 用户等待全部生成后才看到结果 |
| Chitchat也走LLM | "你好"这类查询也要调Router |

---

## 2. 目标

- **成本优化**: 降低40-60% LLM调用成本
- **延迟优化**: 首token响应时间降低50%+
- **体验优化**: 支持流式输出，渐进显示结果

---

## 3. 方案对比

### 方案A: P1快速优化（推荐先做）
**改动范围**: 配置 + 小范围代码调整  
**预期收益**: 成本降低40-60%，延迟降低30%  
**工期**: 1-2天

| 优化项 | 实现方式 | 预期效果 |
|--------|---------|---------|
| Router结果缓存 | 基于query hash缓存5-10分钟 | 减少30-50% Router调用 |
| Chitchat短路前置 | 规则匹配前置，完全跳过Router LLM | 问候类查询0 LLM调用 |
| 模型分级 | Router用轻量模型，Synthesizer用强模型 | Router成本降低60%+ |

**代码改动**:
- `backend/app/services/planner.py`: 增加缓存逻辑
- `backend/app/services/agent.py`: Chitchat规则前置
- `backend/app/config.py`: 新增`router_model`配置

**风险**: 低，可回滚

---

### 方案B: 流式输出优化
**改动范围**: API层 + 前端  
**预期收益**: 首token延迟降低50%+，用户体验大幅提升  
**工期**: 3-5天

**实现要点**:
1. 后端改`stream: true`，支持SSE
2. 前端支持渐进显示
3. Router仍走非流式（快），Synthesizer走流式

**代码改动**:
- `backend/app/api/routes.py`: 新增stream端点
- `backend/app/services/agent.py`: `_synthesize_with_model`支持stream
- `frontend/...`: 支持SSE解析和渐进渲染

**风险**: 中，需前后端联调

---

### 方案C: 架构重构（LangGraph）
**改动范围**: 核心Agent流程重写  
**预期收益**: 更灵活的流程编排，长期可维护性  
**工期**: 2-3周

**实现要点**:
1. 用LangGraph替代手写`agent_graph_nodes.py`
2. 支持条件分支、并行执行、状态持久化
3. 可视化调试

**代码改动**:
- 新增`backend/app/services/agent_graph_v2.py`
- 重写流程节点
- 迁移测试

**风险**: 高，可能影响现有功能稳定性

---

### 方案D: 单LLM模式（激进优化）
**改动范围**: 核心流程重新设计  
**预期收益**: 成本降低50%（1次调用替代2次）  
**工期**: 1-2周

**实现要点**:
1. 设计新prompt：同时输出路由决策和回答
2. 简化的输出格式（JSON with answer）
3. 对复杂查询可能需要fallback到2次调用

**风险**: 高，需大量测试验证准确率

---

## 4. 执行路线（已确认）

**路线**: C → D → A → B

### Phase 1: 方案C - LangGraph重构（2-3周）
- 彻底重写Agent流程，建立现代化架构基础
- 为后续优化打下可维护、可扩展的底子

### Phase 2: 方案D - 单LLM模式（1-2周）
- 在LangGraph架构上实现单LLM调用
- 成本优化核心：2次调用 → 1次调用

### Phase 3: 方案A - P1快速优化（1-2天）
- 在稳定架构上做精细调优
- Router缓存、Chitchat短路、模型分级

### Phase 4: 方案B - 流式输出（3-5天）
- 最终用户体验优化
- 渐进显示、降低感知延迟
